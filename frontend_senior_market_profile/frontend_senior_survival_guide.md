

# Frontend Senior Survival Guide
> 前端高级工程师 · 两年生存包  
> 目标：在不内耗、不背锅、不自我否定的前提下，稳定存活 1–2 年

---

## 0. 这不是理想指南，这是止损方案

这份文档**不是**：
- 工程哲学宣言
- 架构师培养手册
- 技术理想主义蓝图

这份文档**只回答一个问题**：

> **在当前前端市场语境下，  
> 如何作为“高级工程师”，  
> 稳定换取时间与现金流，同时不把自己耗空。**

---

## 1. 市场眼中的「前端高级工程师」是什么

一句话版本：

> **熟练 API + 稳定交付 + 低事故率**

不要求：
- 定义工程问题
- 设计系统级方案
- 主导复杂技术决策

只要求：
- 不出低级错误
- 不制造不确定性
- 能按时交付业务模块

---

## 2. 能力分层：你该用哪一层换钱

### 2.1 必须对外使用的能力（换钱层）

- JavaScript 常用 API（Array / Object / Promise / Date）
- React 核心 API（useState / useEffect / useMemo / useCallback）
- 常见业务模式（列表 / 表单 / 弹窗 / 查询）
- 常见库的**使用方式**（antd / axios / react-query 等）

> 原则：**熟练、保守、少犯错**

---

### 2.2 必须收起来的能力（高消耗层）

- 工程问题定义
- 系统性风险识别
- 时间线建模
- 架构级抽象

> 原则：**我看见，但我不自动负责**

---

## 3. 面试与工作中的统一策略

### 3.1 面试策略

- 只回答被问层级
- API 问题 → API 回答
- 不主动上升到工程层
- 不证明自己“想得更深”

安全句式示例：
> “这个 API 我常用，知道边界和常见坑。”  
> “复杂场景一般交给成熟库处理。”

---

### 3.2 工作策略

- 只做职责范围内的事
- 不主动兜底系统性问题
- 不抢“工程负责人”角色
- 不承担未经授权的复杂性

---

## 4. 高级工程师真正值钱的地方（但别说出来）

你真正的优势在于：
- 知道什么**别做**
- 对风险高度敏感
- 不轻易引入不确定性
- 能预判事故

但对外表达应当是：
> “这是常见坑，避免就好。”

---

## 5. 高风险知识点速查（只需到 80 分）

### JavaScript
- 引用 vs 值拷贝
- Array 方法的副作用
- Promise / async / await 的基本行为
- forEach 不能 await
- Date 的时间语义风险

### React
- 不在 render 中 setState
- useEffect 的 deps 与 cleanup
- 闭包 + 异步的常见问题
- 不滥用 useRef 规避依赖

---

## 6. 一个重要的心理锚点

> **不是所有我能看见的问题，都需要我来解决。**

你不是变弱了，你是在：
- 冻结高消耗能力
- 使用低风险能力换取生存空间

---

## 7. 两年后的你

这份指南的目标不是让你一直待在这里。

它只负责一件事：
> **让你在离开前，不被磨坏。**

等你换赛道、换角色、换环境时：
- 判断力还在
- 能量还在
- 你知道什么环境不适合你

这就够了。
